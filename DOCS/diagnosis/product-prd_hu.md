# 作业提交检查助手 - PRD

## 1. 用户和场景

### 1.1 目标用户

| 用户角色 | 职责 | 使用频率 |
|---------|------|---------|
| 课程导师/助教 | 检查学生提交的 zip 包是否符合课程要求 | 每作业周期1-2次 |
| 课程运营 | 批量处理多个学生的作业提交 | 每作业周期多次 |

### 1.2 使用场景

```
┌─────────────────────────────────────────────────────────────────┐
│                      作业提交流程                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   学生                                     导师/助教               │
│     │                                        │                  │
│     ▼                                        │                  │
│  ┌──────────────┐                           │                  │
│  │ 1. 打包作业   │                           │                  │
│  │   zip 压缩   │                           │                  │
│  └──────┬───────┘                           │                  │
│         │                                   │                  │
│         ▼                                   │                  │
│  ┌──────────────┐                           │                  │
│  │ 2. 提交 zip   │ ──────────────────────────▶                   │
│  └──────────────┘                           │                  │
│                                            │                  │
│                                            ▼                  │
│                                   ┌──────────────────┐          │
│                                   │ 3. 运行检查工具   │          │
│                                   │ 检查作业完整性    │          │
│                                   └────────┬─────────┘          │
│                                            │                    │
│                                            ▼                    │
│                                   ┌──────────────────┐           │
│                                   │ 4. 查看检查报告   │           │
│                                   │ PASS/WARNING/    │           │
│                                   │ BLOCKED          │           │
│                                   └──────────────────┘           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 用户痛点

- **手动检查繁琐**：每个学生的作业包需要逐一检查多个目录和文件
- **标准不统一**：不同检查人员对"合格"的理解可能不一致
- **容易遗漏**：人工检查容易忽略某些必需文件或质量要求

---

## 2. 目标

### 2.1 核心目标

开发一个基于 Bash 的 CLI 工具，自动检查学生提交的 zip 包内容是否符合课程要求。

### 2.2 功能目标

| 编号 | 功能描述 | 优先级 |
|-----|---------|--------|
| F1 | 检查 RAG 助手开发模块（`rag-assistant/`）的文件完整性 | P0 |
| F2 | 检查 Bug 修复实验模块（`bug-fix-lab/`）的文件完整性 | P0 |
| F3 | 验证 ai-log.md 的格式质量和记录数量 | P1 |
| F4 | 验证 README.md 的可复现性（包含安装/运行/测试命令） | P1 |
| F5 | 验证 reflection.md 的内容质量（长度≥10行、无空洞表述） | P1 |
| F6 | 输出结构化的检查报告，包含 PASS/WARNING/BLOCKED 评级 | P0 |

### 2.3 交付物标准

| 作业模块 | 目录 | 必需文件 |
|---------|------|---------|
| RAG 助手开发 | `rag-assistant/` | src/main.py, src/retrieve.py, src/answer.py, data/course-faq.md, docs/spec.md, docs/design.md, docs/test-record.md, README.md, ai-log.md, reflection.md |
| Bug 修复实验 | `bug-fix-lab/` | reproduction.md, root-cause.md, fix-record.md |

---

## 3. 非目标

以下功能不在本项目范围内：

| 编号 | 非目标 | 说明 |
|-----|-------|------|
| N1 | 不修改学生作业 | 只做检查，不做修改或重新打包 |
| N2 | 不做文件内容深度检查 | 只检查文件存在性，不检查代码逻辑或内容质量 |
| N3 | 不做自动修复 | 发现问题只报告，不自动修复 |
| N4 | 不做自动提醒 | 不发送通知或邮件 |
| N5 | 不做多用户并发支持 | 单次检查一个学生的作业包 |
| N6 | 不做云端同步 | 检查结果只输出到终端或文件 |

## 4. 每条需求验收

### 4.1 目录结构检查

| 断点层级 | 检查内容 | 通过标准 | 失败级别 |
|---------|---------|---------|---------|
| L1 | 目录存在性 | `rag-assistant/`, `bug-fix-lab/` 存在 | BLOCKED |
| L2 | 必需目录 | `src/`, `data/`, `docs/` 存在 | BLOCKED |
| L3 | 必需文件 | 所有 .md 和 .py 文件存在 | BLOCKED |

### 4.2 RAG 助手模块必需文件

| 文件路径 | 检查标准 | 失败级别 |
|---------|---------|---------|
| `rag-assistant/src/main.py` | 文件存在 | BLOCKED |
| `rag-assistant/src/retrieve.py` | 文件存在且非空签名版本 | BLOCKED |
| `rag-assistant/src/answer.py` | 文件存在且非空签名版本 | BLOCKED |
| `rag-assistant/data/course-faq.md` | 文件存在 | BLOCKED |
| `rag-assistant/docs/spec.md` | 文件存在 | BLOCKED |
| `rag-assistant/docs/design.md` | 文件存在 | BLOCKED |
| `rag-assistant/docs/test-record.md` | 文件存在 | BLOCKED |
| `rag-assistant/README.md` | 文件存在 | BLOCKED |
| `rag-assistant/ai-log.md` | 文件存在 | BLOCKED |

### 4.3 Bug 修复模块必需文件

| 文件路径 | 检查标准 | 失败级别 |
|---------|---------|---------|
| `bug-fix-lab/reproduction.md` | 文件存在 | BLOCKED |
| `bug-fix-lab/root-cause.md` | 文件存在 | BLOCKED |
| `bug-fix-lab/fix-record.md` | 文件存在 | BLOCKED |

### 4.4 ai-log.md 质量检查

| 检查项 | 通过标准 | 失败级别 |
|-------|---------|---------|
| 五字段格式 | 包含"目的"、"输入"、"建议"、"人工判断"、"验证" | WARNING |
| 条目数量 | 至少 7 条 `## 第N节` 记录 | WARNING |
| 人工判断内容 | 不能只是"AI说得对"、"同意"等空洞内容 | WARNING |

### 4.5 README.md 可复现性检查

| 检查项 | 通过标准 | 失败级别 |
|-------|---------|---------|
| 安装命令 | 包含 pip install / 安装 / install | WARNING |
| 运行命令 | 包含 python3 / 运行 / run | WARNING |
| 测试命令 | 包含 test / 测试 | WARNING |

### 4.6 reflection.md 内容质量检查

| 检查项 | 通过标准 | 失败级别 |
|-------|---------|---------|
| 内容长度 | 至少 10 行 | WARNING |
| 内容实质 | 不包含"收获很多"、"加油"等空洞表述 | WARNING |

### 4.7 检查结果评级规则

| 评级 | 条件 | 含义 |
|-----|------|------|
| ✅ PASS | BLOCKED=0 且 WARN ≤ 2 | 提交包完整，关键证据文件齐全 |
| ⚠️ WARNING | BLOCKED=0 且 WARN > 2 | 有多项需要改进的内容 |
| ❌ BLOCKED | BLOCKED > 0 | 有关键文件缺失，必须修复 |

### 4.8 验收测试用例

```bash
# 测试场景 1: 完整提交包
$ ./check-submission.sh day1-submission/
========================================
 检查结果汇总
========================================
  ✅ PASS:    0
  ⚠️  WARNING: 0
  ❌ BLOCKED: 0
----------------------------------------
 评级: PASS
 原因: 所有必需文件和目录完整，作业包合格
========================================

# 测试场景 2: 缺少关键文件
$ ./check-submission.sh day1-submission/
========================================
 检查结果汇总
========================================
  ❌ BLOCKED: 3
  ⚠️  WARNING: 2
----------------------------------------
 评级: BLOCKED
 原因: 缺少关键文件: rag-assistant/src/main.py, 
       rag-assistant/docs/spec.md, bug-fix-lab/reproduction.md
========================================

# 测试场景 3: 文件齐全但质量不足
$ ./check-submission.sh day1-submission/
========================================
 检查结果汇总
========================================
  ⚠️  WARNING: 5
----------------------------------------
 评级: WARNING
 原因: ai-log 条目数不足，README 缺少测试命令
========================================
```

## 5. 任务优先级

### 5.1 优先级定义

| 优先级 | 定义 | 交付要求 |

| P0 | 核心功能 | 必须完成，否则工具无法使用 |
| P1 | 重要功能 | 应该完成，影响检查质量 |
| P2 | 增强功能 | 最好完成，提升用户体验 |
