## 1. 项目基线

### 1.1 项目概述

#### 1.1.1 项目背景

智能座舱语音车控系统面临三大核心挑战：

1. **多目标复杂性**：用户一句复合指令可能包含多个控制目标（如"打开车窗、关空调、调低座椅加热"）
2. **参数模糊性**：用户口语表达天然存在模糊性（如"留条缝"、"凉一点"、"稍微高点"）
3. **动态车况**：车辆状态（车速、挡位、电量、故障码）实时变化，安全边界随之动态调整

#### 1.1.2 项目定位

构建任务编排与安全校验体系，实现：
- **安全**：越权操作零容忍，高风险操作强制确认
- **可控**：多任务执行可编排、可中断、可回滚
- **可追溯**：全链路日志记录，支持完整审计回溯

#### 1.1.3 核心约束

| 约束类型 | 具体要求 |
|---------|---------|
| 车辆状态约束 | 行驶状态、能源状态、故障状态实时校验 |
| 身份权限约束 | 车主>主驾>乘客>临时访客，高等级操作不可越权 |
| 安全合规约束 | 符合道路交通安全法规，满足车规功能安全要求 |
| 流程可追溯约束 | 指令输入→拆解→校验→确认→执行全链路留痕 |
| 驾驶场景约束 | 行驶状态下确认交互极简化，优先语音交互 |

### 1.3 质量等级

基于安全维度的质量等级定义，**安全为最高优先级**。

#### 1.3.1 质量等级表

| 等级 | 条件 | 含义 |
|------|------|------|
| ✅ PASS | 安全指标100%达标 + 功能指标≥95%达标 | 可发布 |
| ⚠️ WARNING | 安全指标100%达标 + 功能指标≥90%达标 | 有优化项，评审后决定 |
| ❌ BLOCKED | 安全指标未达100% | 存在安全风险，禁止发布 |

#### 1.3.2 安全指标要求

| 指标 | 要求 | 说明 |
|------|------|------|
| 越权操作拦截率 | 100% | 无任何越权执行的逻辑漏洞 |
| 高风险操作确认覆盖率 | 100% | 所有风险操作必须经过明确人工确认 |
| 违规状态操作拦截率 | 100% | 车辆状态不符合执行条件时全部拦截 |
| 安全责任事故 | 0 | 系统上线后零因逻辑漏洞导致的安全事故 |

| 指标 | 要求 |

- 约束明确：无歧义的硬性约束定义

### 2.3 安全架构师角色

#### 2.3.1 职责定义

安全架构师是本项目新增的核心角色，负责安全策略设计与风险评估，确保系统在安全红线内运行。

#### 2.3.2 核心工作

1. **权限模型设计**：角色分级、权限边界、越权处理策略
2. **风险等级划分**：操作风险分类（高/中/低）、确认机制要求
3. **安全校验规则**：车况校验规则、互斥操作检测、依赖关系校验
4. **应急响应策略**：异常情况下的安全降级机制

#### 2.3.3 输入

- `docs/spec.md`
- 业务场景描述
- 车辆状态模型
- 监管合规要求（ISO 26262、道路交通安全法规）

#### 2.3.4 输出

- `docs/safety-strategy.md`：安全策略文档
  - 权限模型定义
  - 风险等级划分表
  - 安全校验规则集
  - 越权拦截处理流程
  - 安全降级策略

#### 2.3.5 质量要求

- 权限覆盖：覆盖所有角色、所有操作
- 风险识别：高风险操作识别率100%
- 校验完备：安全校验规则无遗漏

  - 交互流程图
  - 确认机制设计
  - 反馈话术规范

#### 2.4.5 质量要求

### 3.2 作业流转阶段定义

| 阶段 | 描述 | 责任角色 | 产出 |

| 任务拆解 | 拆分复合指令为独立子任务 | 研发（编排引擎） | 任务列表 |
| 安全校验 | 权限、车况、风险三重校验 | 研发（校验模块） | 校验结果 |
| 确认交互 | 模糊参数确认、高风险确认、越权拒绝 | 研发（交互模块） | 用户响应 |
| 执行调度 | 按序执行任务，处理异常 | 研发（执行模块） | 执行结果 |
| 日志记录 | 全链路留痕 | 研发（日志模块） | 完整日志 |

### 4.1 安全门禁规则

安全门禁是项目发布的必要条件，任何安全指标未达标均不得发布。

#### 4.1.1 安全门禁检查表

| 检查项 | 要求 | 检查方法 | 未达标处理 |
|--------|------|---------|-----------|
| 越权拦截率 | 100% | 安全边界测试套件 | BLOCKED，禁止发布 |
| 高风险操作确认覆盖率 | 100% | 测试用例覆盖率统计 | BLOCKED，禁止发布 |
| 违规状态操作拦截率 | 100% | 车况校验测试套件 | BLOCKED，禁止发布 |
| 安全校验逻辑漏洞 | 0 | 代码审查 + 渗透测试 | BLOCKED，禁止发布 |

#### 4.1.2 安全门禁触发条件

- 每个版本发布前必须通过安全门禁检查
- 新增高风险操作类型必须重新评估安全门禁
- 安全架构师签字确认后方可进入下一阶段
